# 🗺️ Roadmap — udsu knowledge base (UKB)

## ✅ Этап 1. Подготовка и проектирование

### 1.1. Концепт-документ  
- ✍️ Сформировать идею проекта и цели (готово ✅).  
- Описать ключевые фишки, проблемы, которые решает, и целевую аудиторию.  

### 1.2. Техническое задание (ТЗ)  
- Составить список **функциональных** и **нефункциональных требований**.  
- Определить **структуру репозитория** и формат хранения данных.  
- Прописать **правила оформления и именования файлов**.  
- Задать стандарты для README, описаний курсов и предметов.  

### 1.3. Подбор инструментов  
- Определить платформу (GitHub).  
- Выбрать лицензию (MIT).  
- Определить используемые форматы файлов (Markdown, PDF, Code, etc).  
- Настроить базовые интеграции (GitHub Pages, Actions при необходимости).  

---

## 🧩 Этап 2. Разработка структуры и инфраструктуры

### 2.1. Создание репозитория  
- Создать основной репозиторий **udsu knowledge base** на GitHub.  
- Добавить базовые файлы:  
  - `README.md`  
  - `LICENSE`  
  - `.gitignore`  
  - `CONTRIBUTING.md`  
  - `CODE_OF_CONDUCT.md` (по желанию)  

### 2.2. Структура каталогов  
- Создать дерево директорий (по курсам, семестрам, предметам).  
- Добавить пример одного-двух предметов с тестовыми файлами.  
- Настроить README внутри каждой папки (описание предмета и структуры).  

### 2.3. GitHub Pages (опционально)  
- Настроить **витрину сайта** с интерфейсом для просмотра и поиска материалов.  
- Добавить автоматическую генерацию сайта из Markdown файлов (например, через MkDocs).  

---

## 📘 Этап 3. Документация и правила

### 3.1. CONTRIBUTING.md  
- Описать, как добавлять материалы (пошагово).  
- Задать формат имён файлов, README и папок.  
- Прописать пример Pull Request.  

### 3.2. CODE_OF_CONDUCT.md  
- Добавить короткий свод правил общения и поведения участников.  

### 3.3. FAQ и README расширенный  
- Добавить ответы на частые вопросы:  
  - “Как добавить файл?”  
  - “Что можно, а что нельзя выкладывать?”  
  - “Как оформить отчёт или лекцию?”  

---

## 📁 Этап 4. Наполнение и тестирование

### 4.1. Пилотное наполнение  
- Добавить материалы по нескольким предметам (1–2 курса).  
- Проверить структуру, читаемость и навигацию.  
- Исправить, если что-то неудобно.  

### 4.2. Тестирование с другими студентами  
- Попросить 2–3 человек попробовать найти нужные файлы,  
  добавить свои материалы через Pull Request.  
- Собрать фидбэк и улучшить структуру.  

---

## 🚀 Этап 5. Публикация и продвижение

### 5.1. Публичный запуск  
- Сделать пост/анонс в чатах потока или группах VK.  
- Добавить краткую инструкцию для тех, кто хочет помочь.  

### 5.2. Обратная связь и улучшения  
- Создать раздел **Issues** на GitHub для предложений.  
- Добавлять новые функции/разделы по мере роста базы.  

### 5.3. (Опционально) AI и поиск  
- Реализовать интеллектуальный поиск по содержимому файлов (через LLM API).  
- Добавить теги и категории, улучшить GitHub Pages-витрину.  

---

## 🏁 Этап 6. Поддержка и развитие

- Регулярное обновление материалов (1 раз в семестр).  
- Модерация новых Pull Request.  
- Добавление гайдов, шаблонов и вспомогательных инструментов.  
- Подготовка к масштабированию (например, отдельные репозитории по направлениям).
